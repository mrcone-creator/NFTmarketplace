- 直近5件の購入記録を表示（NFT名 / トークンID / ウォレット）

## 💎 NFT設定

サイドバーの **「💎 NFT設定」** から。

ここではNFTの発行枚数・価格・APRを管理者が自由に変更できます。

### 設定項目

各NFT（Silver / Gold / Platina）に対して以下を設定できます：

| 項目 | デフォルト値 | 説明 |
|------|------------|------|
| 最大発行枚数 | Silver:100 / Gold:100 / Platina:50 | 販売可能な最大枚数 |
| 価格 (¥) | Silver:500,000 / Gold:1,000,000 / Platina:2,000,000 | 日本円での販売価格 |
| APR (%) | Silver:18 / Gold:30 / Platina:36 | 年利率 |

> ⚠️ **注意**: 最大発行枚数を**現在の販売済み枚数より少ない値に設定することはできません**（バリデーションで防止）

### 計算プレビュー

設定値を変更すると自動的に以下が計算・表示されます：

- **ETH相当**: 購入価格のETH換算（1 ETH = 設定レート）
- **四半期報酬**: `購入価格(¥) × APR ÷ 4 ÷ USDT換算レート`
- **年間報酬**: 四半期報酬 × 4

### レート設定（デモ）

| レート | デフォルト | 説明 |
|--------|----------|------|
| 1 ETH = ¥ | 400,000 | ETHの日本円換算レート |
| 1 USDT = ¥ | 150 | USDTの日本円換算レート |

### 保存手順

1. 各NFTの設定値を入力
2. **「💾 設定を保存」** ボタンをクリック
3. ✅ トースト通知「NFT設定を保存しました」で完了確認

## 📋 購入履歴

サイドバーの **「📋 購入履歴」** から。

全ユーザーの購入記録を一覧で確認できます。

### 検索・フィルタリング

| 機能 | 説明 |
|------|------|
| 🔍 テキスト検索 | ウォレットアドレス / トークンID / NFT名で絞り込み |
| NFTタイプフィルター | 全NFT / Silver / Gold / Platina で切り替え |

### 一覧の表示項目

| 列 | 説明 |
|----|------|
| NFT | NFT種類と名前 |
| トークンID | 固有のトークン識別子（例: `SILVER-001`） |
| ウォレット | 購入者のウォレットアドレス（短縮表示） |
| 購入日 | 購入日時 |
| 購入金額 | 日本円 / ETH |
| APR | 適用APR |
| 次回報酬日 | 次の四半期報酬配布日（⚠️ マークは期日超過） |
| ステータス | 🔒 運用中 / 📦 未運用 |
| 配布済み | 配布完了回数 / 全回数 |

### CSV出力

**「📥 CSV出力」** ボタンをクリックすると全購入データをCSV形式でダウンロードできます。

出力項目: NFT種類・トークンID・ウォレット・購入日・購入金額・APR・四半期報酬・ステータス・配布回数

## 💰 報酬配布

サイドバーの **「💰 報酬配布」** から。

四半期ごとのUSDT報酬を管理者が手動で配布します。

### 統計カード（4項目）

| カード | 説明 |
|-------|------|
| ⏳ 配布待ち件数 | 報酬発生日を迎えた未配布の件数 |
| 💵 配布待ち合計 | 未配布報酬のUSDT合計額 |
| ✅ 配布済み件数 | これまでの配布完了件数 |
| 💰 配布済み合計 | 配布済みのUSDT合計額 |

### 配布待ちリスト

報酬発生日（購入日から3ヶ月ごと）を迎えたステーキング中のNFTが一覧表示されます。

| 列 | 説明 |
|----|------|
| NFT | NFT種類と名前 |
| トークンID | 固有トークンID |
| ウォレット | 配布先ウォレットアドレス |
| 報酬発生日 | 報酬の発生日 |
| 報酬額 (USDT) | 配布するUSDT額 |
| 操作 | 「💰 配布実行」ボタン |

### 配布方法

#### 1件ずつ配布

各行の **「💰 配布実行」** ボタンをクリック → 該当NFTに報酬が配布されます。

#### 全件一括配布

**「🚀 全件配布実行」** ボタンをクリック → 配布待ち全件を一括で配布します。

> ✅ 配布完了後、「配布済み履歴」テーブルに記録が追加されます。

### 配布済み履歴

| 列 | 説明 |
|----|------|
| NFT | NFT種類・名前・トークンID |
| ウォレット | 配布先ウォレットアドレス |
| 報酬発生日 | 報酬の発生日付 |
| 配布日時 | 管理者が配布を実行した日時 |
| 金額 (USDT) | 配布したUSDT額 |

### 報酬計算式（参考）

```
四半期 USDT 報酬 = 購入価格(¥) × APR(%) ÷ 4 ÷ USDT換算レート(¥)

例) Silver NFT:  ¥500,000 × 18% ÷ 4 ÷ 150 = 375 USDT
例) Gold NFT:    ¥1,000,000 × 30% ÷ 4 ÷ 150 = 500 USDT
例) Platina NFT: ¥2,000,000 × 36% ÷ 4 ÷ 150 = 1,200 USDT
```

## ⚙️ 設定

サイドバーの **「⚙️ 設定」** から。

### データ管理

| 操作 | 説明 |
|------|------|
| 購入データのリセット | 全ての購入・ステーキングデータを削除（**元に戻せません**） |
| 在庫数のリセット | 各NFTの販売済み枚数を0にリセット |

> ⚠️ データリセット操作は確認ダイアログが表示されます。慎重に操作してください。

---

## 🗄️ データ構造

全データはブラウザの **localStorage** に保存されます。

| ストレージキー | 内容 |
|-------------|------|
| `nftm_purchases` | 全ユーザーの購入記録（NFT種類・ウォレット・報酬日・配布履歴） |
| `nftm_nft_supply` | 各NFTの販売済み枚数 |
| `nftm_nft_config` | 管理者が設定したNFT設定値 |
| `nftm_wallet` | ユーザーの接続ウォレット情報 |
| `nftm_activity` | 購入アクティビティログ |

> ⚠️ ブラウザのキャッシュをクリアするとデータが消えます。  
> 本番環境ではデータベース（Firebase / Supabase 等）の使用を推奨します。

## 📋 運用フロー（報酬配布の流れ）

```
1. ユーザーがNFTを購入
        ↓
2. ユーザーがステーキングを開始
        ↓
3. 購入日から3ヶ月後 → 報酬発生日
        ↓
4. 管理者が「報酬配布」ページで配布待ちリストを確認
        ↓
5. 「配布実行」または「全件配布実行」をクリック
        ↓
6. ユーザーのウォレットアドレスにUSDTを送金
   （実際の送金作業は管理者が手動で対応）
        ↓
7. 配布済み履歴に記録が追加される
```

## 🔒 セキュリティに関して

現バージョンではフロントエンドのみの実装です。  
本番環境への移行時は以下を追加してください：

1. **管理者認証** — JWT / OAuth / パスワード認証

3. **スマートコントラクト連携** — ERC-721 / ERC-1155
4. **バックエンドAPI** — Node.js / Express / Python
5. **データベース** — PostgreSQL / MongoDB / Firebase
6. **実際のUSDT送金** — ウォレット連携 / 自動送金スマートコントラクト
